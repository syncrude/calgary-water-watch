Data Sources & Methods

← Back to Dashboard

This page documents where the dashboard gets its data, how it's queried, and how displayed metrics are calculated.

Primary live data source

The dashboard uses the City of Calgary CEAS daily feed hosted as an ArcGIS FeatureService. We query table 15 (CEAS_EXT_WATERDEMAND_DAILY) for fields of interest:

  • METRIC_DATE — measurement date
  • TOTAL_CALGARY_ML — total daily system production (Million Litres per Day, MLD)

Sample REST query used to fetch the latest record (returns JSON):

https://services1.arcgis.com/.../FeatureServer/15/query?where=TOTAL_CALGARY_ML%20IS%20NOT%20NULL&outFields=METRIC_DATE,TOTAL_CALGARY_ML&orderByFields=METRIC_DATE%20DESC&resultRecordCount=1&f=json

Historical time series (3 years)

For the trend chart we request all daily rows since today minus 3 years. The ArcGIS endpoint paginates results; the code pages with resultOffset + resultRecordCount (batch size = 1000) and concatenates features[].attributes.

Fallback data

If the live ArcGIS fetch fails (network, CORS, or the service is unavailable), the dashboard will try a repo-local snapshot file (ceas_daily_fallback.json), if present and sufficiently large. If that's missing or too small, the UI synthesizes a deterministic 3‑year daily series client-side so the charts remain usable for demos and offline testing.

Calculations & sector splits

The dashboard computes derived statistics from TOTAL_CALGARY_ML as follows:

  • Leaks: 22% of TOTAL_CALGARY_ML (baseline infrastructure loss)
  • Usage (remaining 78%) split into:
    • Citizens (residential): 60% of the usage
    • Industry (ICI): 31% of the usage
    • Regional partners: 8% of the usage

Example (pseudocode):

leaks = total * 0.22
usage = total * 0.78
citizens = usage * 0.60
industry = usage * 0.31
regional = usage * 0.08

Dome of Waste Water (visual)

The "Dome of Waste Water" uses a Saddledome capacity of 330 ML. The page calculates:

  • Number of domes = leaks / 330
  • Percent of a dome = (leaks / 330) * 100
  • Litres = leaks * 1,000,000

Notes on data quality and provenance

  • The original Socrata dataset referenced earlier (vax6-p8ay) was missing (404). That is why we use the ArcGIS CEAS feed as the authoritative source.
  • Metric date strings are passed through as provided by the service (often YYYY/MM/DD); the UI normalizes display to YYYY-MM-DD.
  • All derived numbers are shown with one decimal point and are approximate for readability.

How to run and test locally

  1. Start a local HTTP server in the repo root (the site requires serving over HTTP to fetch local files): 
    python3 -m http.server 8000
# or
npx http-server -p 8000
  2. Open http://localhost:8000/index.html and click "Refresh live data" (or disable network to force fallback).
  3. To inspect the raw ArcGIS responses, use curl or your browser network tools with the sample URL above.
